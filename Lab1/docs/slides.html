Three Modes

ModeWhat It DoesWhen to Use It
AskRead-only Q&A"What does this function do?"
PlanAI proposes a plan; you review first"How should I structure this project?"
AgentAI reads, writes, and runs code"Build me a scatter plot"

Think of it as a spectrum:

Ask = just talking  ·  Plan = thinking together  ·  Agent = doing

To learn about models and compare their capabilities, use arena.ai.

6

Choosing the Right Model

Models are a tradeoff between thinking power, speed, and credit cost.

ModelWhen to UseWhy
Opus (Claude 4.6)Complex planning, tough debuggingBest reasoning but very expensive; burns credits quickly
GPT-5.4Balanced tasks, coding + thinkingStrong all-around compromise
Sonnet (Claude 4.6)Fast coding, iterationGreat for execution at lower cost
Codex 5.3Writing code, autocompleteOptimized for coding throughput
Auto / Cursor modelsDaily agent work, easy tasksVery cheap (often effectively free) and credit-friendly

Rule of thumb: use cheap models in Agent mode for easy execution tasks (Composer/Cursor models), and use advanced models in Plan mode for complex reasoning (Opus, GPT-5.4).

Key insight: do not stick to one model. Switching strategically is crucial for both quality and credits.

Use strong models for:

  • Planning projects
  • Understanding errors
  • Designing analysis
  • Interpreting results

Use fast models for:

  • Writing code
  • Editing files
  • Quick fixes
  • Repetitive tasks

Example: Opus → "Help me design a dataset and research question" → Sonnet → "Write Python to simulate data" → Opus → "Interpret results"

Key insight: switch models often. Good workflow = not sticking to one model.

Designing the Analysis

Using Plan Mode

12

Switch to Plan Mode

Change the mode toggle from Agent to Plan.

In Plan mode, the AI will propose a plan but will not change any files.

This is where you think before you code.

This is also where you would use a strong model (Opus / GPT-5.4), since planning requires deep reasoning.

13

Write Your Plan

In Plan mode, describe the analysis you want to run. Here is an example:

I want to study how technological progress affects
employment across industries. Plan a Python analysis that:
1. Generates synthetic panel data for 500 firms over 10
   years with: R&D spending, automation adoption,
   employment, and industry sector
2. Produces summary statistics by sector
3. Runs a fixed-effects regression of employment on
   automation adoption, controlling for R&D and sector
4. Creates a scatter plot of automation vs. employment
   and a bar chart of effects by industry
Save data to data/, scripts to code/, figures to figures/

Now it's your turn. Write a plan for your own project. Describe your research question, the data you need, and what analysis you want to run.

Click to start: 5:00
14

Read the Plan

The AI will produce a structured plan. Take a moment to:

  • Read it carefully. Does the approach make sense?
  • Ask follow-up questions in chat (still in Plan mode)
  • Discuss with your neighbor. What would you change?

Notice: no files were created or changed. Plan mode is read-only.

This mirrors real research workflow: plan first, code second.

17

Exercise A: Inline Editing

Try Ctrl+K (inline edit):

  1. Open a generated .py file
  2. Select a few lines of code (e.g., plotting code)
  3. Press Ctrl+K
  4. Type: Add a title and axis labels, use a colorblind-friendly palette
  5. Press Enter

This is how you make quick, targeted changes without going to the chat panel.

18

Exercise B: Ask Mode for Learning

Switch to Ask mode. Highlight the regression code and type:

Explain what this regression is doing
and interpret the coefficients.

The AI will explain the code without changing anything.

This is how you use Cursor to learn, not just produce.
